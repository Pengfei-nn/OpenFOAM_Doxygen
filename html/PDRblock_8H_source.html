29 Description
30  A single block x-y-z rectilinear mesh addressable as i,j,k with
31  simplified creation. Some of the input is similar to blockMeshDict,
32  but since this specialization is for a single-block that is aligned
33  with the x-y-z directions, it provides a different means of specifying
34  the mesh.
35 
36  Dictionary controls
37  \table
38  Property | Description | Required | Default
39  x | X-direction grid specification | yes |
40  y | Y-direction grid specification | yes |
41  z | Z-direction grid specification | yes |
42  scale | Point scaling | no | 1.0
43  boundary | Boundary patches | yes |
44  defaultPatch | Default patch specification | no |
45  \endtable
46 
47  Grid coordinate controls
48  \table
49  Property| Description | Required | Default
50  points | Locations defining the mesh segment | yes |
51  nCells | Divisions per mesh segment | yes |
52  ratios | Expansion ratio (end/start) per segment | no | uniform
53  \endtable
54 
55  The expansion ratios are defined as per blockMesh and represent the ratio
56  of end-size / start-size for the section. A negative value is trapped
57  and treated as its reciprocal.
