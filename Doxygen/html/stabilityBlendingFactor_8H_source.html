32 Description
33  Calculates and outputs the stabilityBlendingFactor to be used by the
34  local blended convection scheme. The output is a surface field weight
35  between 0-1
36 
37  The weight of a blended scheme is given by a function of the blending
38  factor, f:
39 
40  \f[
41  weight = f scheme1 + (1 - f) scheme2
42  \f]
43 
44  The factor is calculated based on six criteria:
45 
46  1) mesh non-orthogonality field
47  2) magnitude of cell centres gradient
48  3) convergence rate of residuals
49  4) faceWeight
50  5) skewness
51  6) Co number
52 
53  The user can enable them individually.
54 
55  For option 1, the following relation is used:
56  \f[
57  fNon =
58  min
59  (
60  max
61  (
62  0.0,
63  (nonOrthogonality - maxNonOrthogonality)
64  /(minNonOrthogonality - maxNonOrthogonality)
65  ),
66  1.0
67  )
68  \f]
69 
70  For option 2, the following relation is used:
71  \f[
72  fMagGradCc =
73  min
74  (
75  max
76  (
77  0.0,
78  (magGradCC - maxGradCc)
79  / (minGradCc - maxGradCc)
80  ),
81  1.0
82  )
83  \f]
84  Note that magGradCC is equal to 3 for ortoghonal meshes
85 
86  For option 3 a PID control is used in order to control residual
87  unbouded fluctuations for individual cells.
88 
89  \f[
90  factor =
91  P*residual
92  + I*residualIntegral
93  + D*residualDifferential
94  \f]
95 
96  where P, I and D are user inputs
97 
98  The following relation is used:
99  \f[
100  fRes = (factor - meanRes)/(maxRes*meanRes);
101  \f]
102 
103  where:
104  meanRes = average(residual)
105  maxRes is an user input
106 
107 
108  fRes will blend more towards one as the cell residual is larger then
109  the domain mean residuals.
110 
111 
112  For option 4 , the following relation is used:
113 
114  \f[
115  ffaceWeight = min
116  (
117  max
118  (
119  0.0,
120  (minFaceWeight - faceWeights)
121  / (minFaceWeight - maxFaceWeight)
122  ),
123  1.0
124  )
125  \f]
126  Note that faceWeights for a orthogonal mesh is 0.5.
127 
128 
129  For option 5 , the following relation is used:
130 
131  \f[
132  fskewness =
133  min
134  (
135  max
136  (
137  0.0,
138  (skewness - maxSkewness)
139  / (minSkewness - maxSkewness)
140  ),
141  1.0
142  )
143  \f]
144 
145 
146  For option 6 , the following relation is used:
147 
148  \f[
149  fCoWeight = min(max((Co - Co1)/(Co2 - Co1), 0), 1)
150  \f]
151  where
152 
153  \vartable
154  Co1 | Courant number below which scheme2 is used
155  Co2 | Courant number above which scheme1 is used
156  \endvartable
157 
158  The final factor is determined by:
159 
160  \f[
161  f = max(fNon, fMagGradCc, fRes, ffaceWeight, fskewness, fCoWeight)
162  \f]
163 
164  An indicator (volume) field, named blendedIndicator is generated if the log
165  flag is on:
166  - 1 represent scheme1 as active,
167  - 0 represent scheme2 as active.
168 
169  Additional reporting is written to the standard output, providing
170  statistics as to the number of cells used by each scheme.
171 
172 Usage
173  Example of function object specification to calculate the blending factor:
174  \verbatim
175  stabilityBlendingFactor1
176  {
177  type stabilityBlendingFactor;
178  libs ("libfieldFunctionObjects.so");
179 
180  log true;
181  writeToFile false;
182 
183  switchNonOrtho yes;
184  switchGradCc no;
185  switchResiduals yes;
186  switchFaceWeight no;
187  switchSkewness no;
188  switchCo no;
189 
190  maxNonOrthogonality 20;
191  minNonOrthogonality 60;
192 
193  maxGradCc 3;
194  minGradCc 4;
195 
196  maxFaceWeight 0.3;
197  minFaceWeight 0.2;
198 
199  maxSkewness 2;
200  minSkewness 3;
201 
202  maxResidual 10;
203 
204  result UBlendingFactor;
205  residual initialResidual:p;
206  P 1.5;
207  I 0;
208  D 0.5;
209 
210  Co1 1;
211  Co2 10;
212  ...
213  ...
214  field U;
215  }
216 
217  Example of function object specification to calculate the residuals used
218  by stabilityBlendingFactor. The following writes 'initialResidual:p'
219  field
220 
221  residuals
222  {
223  type residuals;
224  libs ("libutilityFunctionObjects.so");
225  writeFields true;
226  writeControl outputTime;
227  fields (p);
228  }
229 
230  \endverbatim
231 
232  Where the entries comprise:
233  \table
234  Property | Description | Required | Default value
235  type | Type name: stabilityBlendingFactor | yes |
236 
237  log | Log to standard output | no | yes
238  writeToFile | Log to file | no | yes
239 
240  switchNonOrtho | non-orthogonal method | no | false
241  switchGradCc | cell centre gradient method | no | false
242  switchResiduals | residual evolution method | no | false
243  switchFaceWeight | face weight method | no | false
244  switchSkewness | skewness method | no | false
245  switchCo | Co blended | no | false
246 
247  maxNonOrthogonality| maximum non-orthogonal for scheme2 | no | 20
248  minNonOrthogonality| minimum non-orthogonal for scheme1 | no | 60
249 
250  maxGradCc| maximum gradient for scheme2 | no | 2
251  minGradCc| minimum gradient for scheme1 | no | 4
252 
253  maxResidual| maximum residual-mean ratio for scheme1 | no | 10
254  P | proportional factor for PID | no | 3
255  I | integral factor for PID | no | 0
256  D | differential factor for PID | no | 0.25
257 
258  maxFaceWeight | maximum face weight for scheme1 | no | 0.2
259  minFaceWeight | minimum face weight for scheme2 | no | 0.3
260 
261  maxSkewness | maximum skewness for scheme2 | no | 2
262  minSkewness | minimum skewness for scheme1 | no | 3
263 
264 
265  faceWeight | Name of the faceWeight field | no | faceWeight
266  skewness | Name of the skewness field | no | skewness
267  nonOrthogonality | Name of the non-orthogonal field | no |
268  nonOrthoAngle
269  residual | Name of the residual field | no | initialResidual:p
270  U | Name of the flux field for Co blended | no | U
271 
272 
273  Co1 | Courant number below which scheme2 is used | no | 1
274  Co2 | Courant number above which scheme1 is used | no | 10
275 
276 
277  tolerance | Tolerance for number of blended cells | no | 0.001
278  field | Name of field to evaluate | yes |
279  result | Name of surface field to be used in the localBlended scheme
280  | yes |
281  \endtable
282 
283  The 'log' flag true write the number of cells on each scheme, plus it
284  writes the field name "blendedIndicator".
285 
286  The 'result' entry is the field which is read by the localBlended scheme
287  specified in fvSchemes. This name is determined by the localBlended class.
