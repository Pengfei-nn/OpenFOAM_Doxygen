26 Class
27  Foam::functionObjects::runTimePostPro::volumeFilter
28 
29 Description
30  Visualisation of OpenFOAM volume fields as surface data using
31  a VTK filter cascade.
32 
33 Note
34  Since this filter includes an OpenFOAM/VTK adaptor level,
35  it is ill-suited to mismatches in data parallelization.
36  If OpenFOAM is running in parallel but VTK is not, it would be rather
37  expensive to collect all the data on the master node for this filter.
38  That approach is acceptable for smaller amounts of data, but do not
39  allow for volume meshes.
